Look and feel:
I want to deliver a clean and neat feeling by establishing a user-friendly planning site that is minimalistic.

Research on usability and User Interface:
I would need to consider the following aspects when I make the navigation bar.

1. Accessibility of the buttons: there is an F-shaped pattern when users skim through the site (top and left is viewed first).
2. Uniformity and consistency: the way each button operates should be the same so users would not be confused (single click for all)
3. Flow of user interface: the buttons that direct users to other sections of the site should be accessible on any viewing page without pressing the "back" button

As a result, I am placing the buttons on the very top of the site, which would become a permanent section for all pages.

How to create navigation buttons on Fireworks

The actual graphical buttons are created using rectangular objects.
Once a template is created for one button, all the other ones can be created with ease following the same format.
To distinguish between which buttons the user is trying to access, not accessing, or in the process of accessing,
there are different attributes we can set for the different states it is in. For instance, in the " up state", or initial state, the 3D image will have a "raised" surface as that of a cube.

Similarly, to distinguish a hover, the button will "sink in" and decrease in opacity (the white layer) as demonstrated in the image below.
